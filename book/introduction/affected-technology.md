# เทคโนโลยีที่ได้รับผลกระทบจากคอมพิวเตอร์ควอนตัม

ในอนาคตอันใกล้ หากระบบรหัสลับสำหรับยุคควอนตัม หรือ Post-Quantum Cryptography (PQC) ได้รับการกำหนดเป็นมาตรฐานแล้ว มันจะกลายเป็นมาตรฐานระบบรหัสลับใหม่ที่องค์กรและอุตสาหกรรมทั่วโลกต้องนำมาใช้แทนระบบรหัสลับดั้งเดิม (Classical Cryptography) แอปพลิเคชันและเทคโนโลยีสารสนเทศจำนวนมากจึงจำเป็นต้องได้รับการปรับปรุงเพื่อให้รองรับอัลกอริทึมประเภท PQC ได้อย่างเหมาะสม {cite:p}`nist2024ir8547` โดยเปลี่ยนแปลงนี้จะไม่ใช่เพียงการอัปเกรดทางเทคนิคในระยะสั้น แต่เป็นการปรับโครงสร้างด้านความมั่นคงปลอดภัยในระยะยาวของระบบสารสนเทศทั้งระบบ

การเปลี่ยนผ่านไปสู่ PQC อาจก่อให้เกิดการหยุดชะงักของการดำเนินงาน (Operational Disruption) ขององค์กร โดยเฉพาะองค์กรขนาดใหญ่ที่มีระบบสารสนเทศที่ซับซ้อนและพึ่งพาระบบรหัสลับในหลายชั้น เนื่องจากอัลกอริทึม PQC มีขนาดกุญแจรหัสลับและขนาดลายมือชื่อดิจิทัล (Digital Signature) ที่ใหญ่กว่าอัลกอริทึมดั้งเดิมอย่างมีนัยสำคัญ จึงอาจส่งผลให้ระบบที่มีอยู่เดิมมีประสิทธิภาพลดลง (Performance Degradation) หรือไม่สามารถทำงานได้เลยหากไม่ได้รับการปรับปรุงให้รองรับ

ด้วยเหตุนี้ เทคโนโลยีสารสนเทศต่าง ๆ จึงต้องได้รับการพิจารณา ปรับเปลี่ยน และ ทดสอบอย่างถี่ถ้วน เพื่อรองรับภัยคุกคามจากคอมพิวเตอร์ควอนตัม โดยมีตัวอย่างเทคโนโลยีที่จะได้รับผลกระทบดังต่อไปนี้

## การลงลายมือชื่อบนซอฟต์แวร์
การลงลายมือชื่อบนซอฟต์แวร์ (Code Signing) คือกระบวนการลงลายมือชื่อดิจิทัลบนไฟล์ชุดคำสั่ง (Executables) และแพ็กเกจซอฟต์แวร์ (Software Packages) เพื่อยืนยันตัวตนของผู้พัฒนาและรับรองว่าซอฟต์แวร์ไม่ได้ถูกแก้ไขดัดแปลง กระบวนการนี้มีความสำคัญในการสร้างความเชื่อมั่นในซอฟต์แวร์และป้องกันการแพร่กระจายของซอฟต์แวร์ที่เป็นอันตราย (Malicious Code)

อุปกรณ์ที่ติดตั้งซอฟต์แวร์เหล่านี้ต้องสามารถตรวจสอบลายมือชื่อดิจิทัลได้ ทว่าอุปกรณ์บางชนิดที่ผลิตออกมาแล้วอาจไม่สามารถปรับเปลี่ยนซอฟต์แวร์หรือเฟิร์มแวร์ภายในได้ ทำให้มีความเสี่ยงต่อการถูกโจมตีด้วยคอมพิวเตอร์ควอนตัมในอนาคต ดังนั้น อุปกรณ์ที่ต้องใช้งานเป็นเวลานานควรได้รับการออกแบบให้รองรับการลงลายมือชื่อดิจิทัลที่ทนทานต่อการโจมตีด้วยคอมพิวเตอร์ควอนตัม เช่น มาตรฐาน FIPS 204 และ FIPS 205

## การลงลายมือชื่อและเข้ารหัสลับของอีเมลและเอกสาร
การลงลายมือชื่อดิจิทัลบนอีเมลและเอกสารอิเล็กทรอนิกส์ช่วยยืนยันแหล่งที่มา (Authenticity) และความสมบูรณ์ (Integrity) ของข้อมูล ทำให้ผู้รับมั่นใจได้ว่าข้อความไม่ได้ถูกแก้ไขและมาจากแหล่งที่น่าเชื่อถือ

โพรโทคอลสำหรับลงลายมือชื่อดิจิทัลและเข้ารหัสลับข้อมูล เช่น Secure/Multipurpose Internet Mail Extensions (S/MIME) ใช้ระบบรหัสลับกุญแจสาธารณะ (Public Key Cryptography) ในการลงลายมือชื่อและเข้ารหัสลับแบบปลายทางถึงปลายทาง (End-to-End Encryption) เพื่อยืนยันตัวตนและแลกเปลี่ยนข้อมูล โดยมีความเสี่ยงจากการเก็บเกี่ยวข้อมูลเพื่อถอดรหัสลับในภายหลัง (Harvest Now, Decrypt Later: HNDL) เช่นเดียวกับโพรโทคอลอื่น ๆ ที่กล่าวถึงในหัวข้อถัดไป

## การยืนยันตัวตนผู้ใช้งานและระบบ
ระบบยืนยันตัวตนสำหรับผู้ใช้งาน (User Authentication) และระบบ (Machine Authentication) โดยส่วนมากอาศัยระบบรหัสลับแบบกุญแจอสมมาตร (Asymmetric Key Cryptography) ในการสร้างกุญแจรหัสลับร่วมกัน (Key Establishment) หรือการลงลายมือชื่อดิจิทัล (Digital Signature) เพื่อให้มั่นใจว่าเฉพาะผู้ใช้งานหรือระบบที่ได้รับอนุญาตเท่านั้นที่สามารถเข้าถึงข้อมูลหรือบริการที่สำคัญได้

ระบบยืนยันตัวตนแตกต่างจากการเข้ารหัสลับ (Encryption) ตรงที่ไม่มีความเสี่ยงจากการเก็บเกี่ยวข้อมูลเพื่อถอดรหัสลับในภายหลัง (Harvest Now, Decrypt Later: HNDL) องค์กรจึงยังคงใช้ระบบยืนยันตัวตนด้วยระบบรหัสลับดั้งเดิม (Classical Cryptography) ได้ จนกว่าคอมพิวเตอร์ควอนตัมที่มีขนาดเพียงพอโจมตีระบบรหัสลับดั้งเดิม (CRQC) จะถูกพัฒนาขึ้น

อย่างไรก็ตาม อัลกอริทึมสำหรับการยืนยันตัวตนที่ทนทานต่อการโจมตีด้วยคอมพิวเตอร์ควอนตัมจำเป็นต้องได้รับการปรับปรุงทั้งในฝั่งระบบที่ดำเนินการยืนยันตัวตนและระบบที่ยอมรับการยืนยันตัวตน รวมถึงโครงสร้างพื้นฐานที่สนับสนุน เช่น ระบบโครงสร้างพื้นฐานกุญแจสาธารณะและอุปกรณ์ฮาร์ดแวร์ที่เกี่ยวข้อง

## โพรโทคอลเครือข่าย
โพรโทคอลเครือข่าย เช่น Transport Layer Security (TLS), Secure Shell (SSH) และ Internet Protocol Security (IPsec) กำหนดกฎเกณฑ์สำหรับการแลกเปลี่ยนข้อมูลผ่านเครือข่ายอย่างปลอดภัย แต่ยังคงพึ่งพาระบบรหัสลับดั้งเดิม (Classical Cryptography) ซึ่งมีความเสี่ยงต่อการโจมตีด้วยคอมพิวเตอร์ควอนตัม

โพรโทคอลเหล่านี้จำเป็นต้องได้รับการปรับปรุงให้รองรับอัลกอริทึม PQC โดยเฉพาะในส่วนของกลไกการสร้างกุญแจรหัสลับร่วมกัน (Key Establishment) และการยืนยันตัวตน ซึ่งอาจนำไปสู่การเปลี่ยนแปลงเชิงโครงสร้าง เช่น การรองรับกุญแจที่มีขนาดใหญ่ขึ้น

โพรโทคอลสมัยใหม่มักแยกการใช้งานระหว่างกุญแจระยะยาวสำหรับการยืนยันตัวตนและกุญแจระยะสั้นสำหรับการสร้างกุญแจรหัสลับร่วมกัน เพื่อให้ได้คุณสมบัติ Forward Secrecy อย่างไรก็ตาม กุญแจรหัสลับแบบสมมาตรที่ได้จากกระบวนการดังกล่าวยังมีความเสี่ยงจากการเก็บเกี่ยวข้อมูลเพื่อถอดรหัสลับในภายหลัง (Harvest Now, Decrypt Later: HNDL) การปรับปรุงกลไกการสร้างกุญแจจึงเป็นประเด็นเร่งด่วนกว่าการปรับปรุงอัลกอริทึมยืนยันตัวตน

## ไลบรารีซอฟต์แวร์ระบบรหัสลับ
ไลบรารีซอฟต์แวร์ระบบรหัสลับ (Software Cryptographic Libraries) คือชุดอัลกอริทึมและโพรโทคอลระบบรหัสที่นักพัฒนาใช้สร้างแอปพลิเคชันต่าง ๆ ตัวอย่างที่มีการใช้งานอย่างแพร่หลาย ได้แก่ OpenSSL, BoringSSL, Libsodium และ Java Cryptography Architecture (JCA)

ไลบรารีเหล่านี้จำเป็นต้องได้รับการปรับปรุงให้รองรับอัลกอริทึมประเภท PQC ตามมาตรฐานสากล เพื่อให้นักพัฒนาสามารถนำไปใช้งานได้โดยไม่ต้องพัฒนาอัลกอริทึมที่ซับซ้อนเหล่านี้ด้วยตนเอง นอกจากนี้ ยังต้องผ่านการทดสอบความมั่นคงปลอดภัยอย่างรอบคอบ เพื่อให้มั่นใจว่าปลอดภัยจากการโจมตีช่องทาด้านข้าง (Side-Channel Attacks)

## อุปกรณ์ฮาร์ดแวร์ระบบรหัส
อุปกรณ์ฮาร์ดแวร์ระบบรหัส (Cryptographic Hardware) เช่น Hardware Security Modules (HSMs) และ Trusted Platform Modules (TPMs) มีบทบาทสำคัญในการรักษาความมั่นคงปลอดภัยสำหรับการดำเนินงานด้านระบบรหัสและการจัดเก็บกุญแจที่สำคัญ ครอบคลุมการใช้งานตั้งแต่การรักษาความปลอดภัยโครงสร้างพื้นฐานของเซิร์ฟเวอร์ไปจนถึงการปกป้องกุญแจรหัสลับบนอุปกรณ์ของเจ้าหน้าที่

อุปกรณ์เหล่านี้จำเป็นต้องได้รับการปรับปรุงหรือออกแบบใหม่เพื่อรองรับอัลกอริทึมประเภท PQC ซึ่งต้องใช้กุญแจรหัสลับขนาดใหญ่ขึ้นและต้องการระบบการคำนวณที่แตกต่างจากระบบรหัสลับดั้งเดิม (Classical Cryptography) กระบวนการปรับปรุงอาจครอบคลุมทั้งการเปลี่ยนเฟิร์มแวร์ (Firmware) และฮาร์ดแวร์ เพื่อให้ทนทานต่อการโจมตีจากคอมพิวเตอร์ควอนตัมได้อย่างมีประสิทธิภาพ

## องค์ประกอบด้านโครงสร้างพื้นฐาน
โครงสร้างพื้นฐานกุญแจสาธารณะ (Public Key Infrastructure: PKI) เป็นระบบบริหารจัดการใบรับรองอิเล็กทรอนิกส์และคู่กุญแจสาธารณะ-ส่วนตัว (Public-Private Key Pairs) เพื่อรักษาความมั่นคงปลอดภัยในการสื่อสารและการยืนยันตัวตนในระบบเครือข่าย ประกอบด้วยองค์ประกอบสำคัญ ได้แก่ ระบบผู้ให้บริการออกใบรับรอง (Certification Authority: CA) ระบบเจ้าหน้าที่รับลงทะเบียน (Registration Authority: RA) ระบบบริหารจัดการกุญแจ และบริการไดเรกทอรี (Directory Service)

ระบบ PKI และองค์ประกอบที่เกี่ยวข้องจำเป็นต้องได้รับการปรับปรุงให้รองรับอัลกอริทึมประเภท PQC เพื่อให้สามารถออก (Issue) แจกจ่าย (Distribute) จัดการ (Manage) และเพิกถอน (Revoke) ใบรับรองอิเล็กทรอนิกส์ที่ทนทานต่อการโจมตีจากคอมพิวเตอร์ควอนตัมได้ ทั้งนี้ ระบบ PKI ที่ได้รับการปรับปรุงยังต้องสามารถทำงานร่วมกับระบบรหัสลับดั้งเดิม (Classical Cryptography) ได้ในช่วงเปลี่ยนผ่านสู่ยุคควอนตัม

## แอปพลิเคชันและบริการเทคโนโลยีสารสนเทศ
แอปพลิเคชันและบริการเทคโนโลยีสารสนเทศ (IT Applications and Services) ครอบคลุมซอฟต์แวร์และแพลตฟอร์มที่ใช้งานภายในองค์กร เช่น เว็บแอปพลิเคชัน ฐานข้อมูล บริการคลาวด์ และซอฟต์แวร์บริหารจัดการ ซึ่งล้วนอาศัยระบบรหัสลับในการปกป้องข้อมูล ยืนยันตัวตนผู้ใช้งาน และรับรองความปลอดภัยของธุรกรรม

แอปพลิเคชันและบริการเหล่านี้จำเป็นต้องได้รับการปรับปรุงให้รองรับอัลกอริทึมประเภท PQC สำหรับการเข้ารหัสลับ (Encryption) การถอดรหัสลับ (Decryption) การลงลายมือชื่อดิจิทัล (Digital Signature) และการสร้างกุญแจรหัสลับร่วมกัน (Key Establishment) ซอฟต์แวร์ต้องได้รับการปรับให้รองรับกุญแจขนาดใหญ่ขึ้นและประมวลผลอัลกอริทึม PQC ที่ต้องการทรัพยากรด้านการคำนวณมากขึ้น พร้อมทั้งผ่านการทดสอบความมั่นคงปลอดภัยอย่างเข้มงวด และอาจต้องปรับปรุงอินเทอร์เฟซผู้ใช้งาน (User Interface) ให้รองรับการเปลี่ยนแปลงดังกล่าวด้วย
